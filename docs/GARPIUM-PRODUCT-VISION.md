## 1. Позиционирование

**GARPIUM** — SaaS-платформа нового поколения для корпоративного обучения, адаптации сотрудников, управления знаниями компании, развития персонала и создания внутренних корпоративных академий.

Цель — не «ещё одна LMS для прохождения курсов», а **Corporate Learning Operating System (CLOS)**: единая цифровая среда, где объединены сотрудники, обучение, база знаний, адаптация, регламенты, аналитика и контроль развития персонала.

### Проблема, которую решает платформа

- Потеря знаний внутри компании
- Длительная адаптация новых сотрудников
- Отсутствие контроля обучения
- Разрозненность информации между сервисами

### Ключевая ценность для бизнеса

- Ускорение адаптации сотрудников
- Сохранение знаний внутри компании
- Снижение затрат на обучение
- Контроль развития персонала
- Создание корпоративной академии без сложного внедрения

### Конкурентное преимущество

GARPIUM позиционируется как **единая платформа управления знаниями, адаптацией и корпоративным обучением**, а не как каталог курсов.

## 2. Целевая аудитория

| Сегмент | Потребность |
|---------|-------------|
| Малый бизнес (от 10 чел.) | Быстрый старт обучения без IT-отдела |
| Средний бизнес (20–500) | Структура, отделы, аналитика |
| Корпоративные академии | Собственный университет, сертификация |
| Франшизы и сети | Единые стандарты, white-label |
| Отделы продаж | Быстрая адаптация, KPI обучения |
| Производство, IT, логистика, медицина | Регламенты, переаттестация, compliance |
| HR-отделы | Онбординг, карта компетенций |
| Компании с высокой текучестью | Стандартизированная адаптация |

## 3. Архитектура: Multi-Tenant SaaS

```
Platform (GARPIUM)
  └── Organization / Company (Tenant)
        └── Users → Roles → Departments → Teams
              └── Courses → Learning Paths → Knowledge Base
                    └── Certificates → Analytics → Integrations
```

**Принципы:**

- Каждая компания — изолированное пространство (tenant)
- Данные компаний полностью изолированы друг от друга
- Один вход для всех типов пользователей; роль определяется после авторизации
- Компания может работать как внутренняя академия с white-label элементами

## 4. Регистрация и компания

### 4.1 После регистрации

Пользователь получает обычный аккаунт. В личном кабинете — раздел **«Моя компания»**:

1. **Создать компанию** — с обязательной верификацией
2. **Присоединиться к компании** — по приглашению
3. **Продолжить как частное лицо** — без организации

### 4.2 Верификация компании

Обязательная проверка при создании организации.

**Данные:**

- Название, ОПФ (самозанятый, ИП, ООО, АО, ПАО, НКО, госорганизация)
- ИНН, КПП (при необходимости), ОГРН / ОГРНИП
- Юридическое название и адрес
- Корпоративный email
- Контактное лицо
- Подтверждающие документы
- Официальное подтверждение через **Госуслуги (ЕСИА)**

**Статусы:** `PENDING_VERIFICATION` → `VERIFIED` | `REJECTED`

До верификации корпоративный функционал ограничен.

### 4.3 Приглашения (не постоянные коды компании)

Администратор создаёт **безопасное приглашение**:

| Параметр | Описание |
|----------|----------|
| Уникальный токен | Например `GARP-8H2K-9X5P-Q7L1` или `/invite/8H2K9X5PQ7L1` |
| Срок действия | Опционально |
| Лимит использований | Одноразовое / многоразовое |
| Роль | Employee, Manager, Admin |
| Отдел | Опционально |
| Должность | Опционально |

Приглашение можно **отозвать** в любой момент.

**Подключение по домену:** `@company.com` → автопредложение вступить при регистрации (с подтверждением администратора).

## 5. Роли (обзор)

Детальная модель — в [GARPIUM-RBAC-ACCESS-MODEL.md](./GARPIUM-RBAC-ACCESS-MODEL.md).

### Tenant-роли (внутри компании)

```
Employee → Manager → Company Admin → Company Owner
```

### Platform-роли

| Роль | Назначение |
|------|------------|
| Guest | Публичный сайт, тарифы, документация |
| Garpium Support | Тикеты поддержки |
| Garpium Moderator | Верификация компаний и модерация |
| Super Admin | Полный доступ к платформе |

### Дополнительные роли продукта

| Роль | Назначение |
|------|------------|
| PUBLIC_USER | Частное лицо без компании |
| INSTRUCTOR | Автор курсов, Studio |
| GARPIUM_EMPLOYEE | Внутренний доступ Garpium |

## 6. Модули платформы

### 6.1 Личный кабинет

Персональное пространство пользователя.

**Вкладки:** Главная · Обучение · Профиль · Моя компания · Настройки

**Главная показывает:**

- Прогресс обучения (круговые индикаторы)
- Продолжить обучение (featured course)
- Достижения и активность
- Уведомления, дедлайны, задачи (целевое состояние)

**Профиль:** email, телефон, пароль, Госуслуги — с подтверждением.

### 6.2 Компания (корпоративный кабинет)

| Раздел | Функции |
|--------|---------|
| Обзор | Метрики, статус верификации |
| Сотрудники | Карточки, роли, прогресс, действия |
| Отделы | Иерархия, руководители |
| Курсы | Назначения, каталог компании |
| Программы | Учебные траектории |
| База знаний | Корпоративная Wiki |
| Onboarding | Программы адаптации |
| Приглашения | Создание, отзыв, коды |
| Аналитика | Успеваемость, отделы, audit |
| Интеграции | API, webhooks, CRM/HRM |
| Настройки | Домены, white-label, SSO |

### 6.3 Обучение (LMS-ядро)

- Курсы, модули, уроки
- Программы обучения (learning paths)
- Тестирование, экзамены, практические задания
- Прогресс, баллы, уровни
- Назначение: сотрудник / группа / отдел / вся компания

**Конструктор курсов (блоки):**

текст · видео · изображение · PDF · презентации · тесты · опросы · код · ссылки · вложения

### 6.4 Онбординг

Автоматизированная программа адаптации нового сотрудника.

- Чек-листы, задачи, инструкции
- Привязка курсов и экзаменов
- Контроль прохождения
- Оценка адаптации руководителем

**Ключевое конкурентное преимущество** перед классическими LMS.

### 6.5 База знаний

Корпоративное Wiki (Notion / Confluence-подобное):

- Статьи, инструкции, регламенты, FAQ, шаблоны
- Права доступа по роли и отделу
- Версионирование (целевое состояние)

### 6.6 Сертификаты

После завершения курса — автоматическая выдача.

| Элемент | Описание |
|---------|----------|
| Уникальный номер | Публичная проверка |
| QR-код | Сканирование |
| Страница verify | `/certificates/[number]` |
| Срок действия | Для переаттестации |

### 6.7 Аналитика

| Уровень | Данные |
|---------|--------|
| Employee | Личная успеваемость |
| Manager | Команда / отдел |
| Company Admin | Вся компания, экспорт |
| Company Owner | + бизнес-метрики, ROI |

### 6.8 API и интеграции

Каждая компания: API Key, Secret, Webhooks.

**Интеграции:** CRM, ERP, HRM, Telegram, Slack, Bitrix24, AmoCRM, Jira, GitHub, Google Workspace, Microsoft 365.

### 6.9 Администрирование платформы

- Верификация организаций (Moderator)
- Управление контентом LMS (Super Admin)
- Модерация маркетплейса
- Глобальная аналитика

## 8. Специальный режим Garpium

Организация Garpium (`isGarpium: true`):

- Бесплатный доступ к внутренним курсам, регламентам, тестам
- Стандарты компании для всех сотрудников Garpium

---

## 9. White Label

Компания может:

- Подключить домен / subdomain
- Загрузить логотип, выбрать цвета
- Настроить внешний вид

**Обязательно сохраняется:** `Powered by Garpium` или `Developed by Garpium`

## 10. Безопасность (целевое состояние)

- JWT Access + Refresh Token, HttpOnly Cookies
- RBAC, CSRF Protection
- 2FA, контроль сессий, история входов
- Защита от перебора, audit log
- Журнал действий: user, IP, device, time, action

## 13. UI / UX принципы

**Не изменять без необходимости** — все новые страницы должны выглядеть как естественное продолжение системы.

| Принцип | Реализация в коде |
|---------|-------------------|
| Минимализм, воздух | `--surface-muted`, мягкие границы `--border-soft` |
| Карточная структура | `.lms-card`, `.dash-card`, `.pdata-panel` |
| Без визуального шума | Не злоупотреблять `lms-frame` и жёсткими рамками |
| Единый стиль | `lms-base.css`, `dashboard.css`, `lms-shell.css` |
| Акцент | `--accent` (#e8622a), кнопки `.course-button.is-accent` |
| Типографика | Inter + Montserrat Alternates |
| Три зоны UI | Marketing (`home.css`) · Auth · LMS Shell (`CourseShell`) |

## 14. Масштабирование

Архитектура рассчитана на:

- Тысячи компаний
- Сотни тысяч сотрудников
- Миллионы прохождений курсов

Все сущности проектируются с учётом роста без полного рефакторинга.
