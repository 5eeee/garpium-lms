# Подключение GARPIUM LMS к вашей компании

Документация для партнёров и клиентов GARPIUM: как устроена платформа, как подключить white-label обучение HTML/CSS к своему сайту или продукту.

---

## 1. Что это

**GARPIUM LMS** — корпоративная платформа обучения вёрстке:

- **120 уроков**: 60 HTML + 60 CSS
- Практика с проверкой кода на каждом уроке
- Карта курса (roadmap), глоссарий, мини-игры
- Личный кабинет, прогресс, сертификат
- Админка для модерации учеников и редактирования контента
- **White-label**: ваш бренд + подпись «Обучение предоставлено GARPIUM»
- **Partner API** для интеграции с вашей CRM, сайтом или приложением

---

## 2. Как это устроено

```
┌─────────────────────────────────────────────────────────────┐
│  Ваш поддомен:  acme.lms.garpium.com  (или свой домен)      │
│  ┌───────────────────────────────────────────────────────┐  │
│  │  Шапка: [лого вашей компании]  + навигация            │  │
│  │  Контент: уроки, карта, профиль, глоссарий            │  │
│  │  Футер: «Обучение предоставлено GARPIUM»              │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
         ▲                              ▲
         │                              │
   DNS CNAME                    Partner API (Bearer key)
   на наш сервер                для вашей CRM / сайта
```

### Компоненты

| Компонент | Описание |
|-----------|----------|
| **Веб-платформа** | Next.js приложение, PostgreSQL, авторизация |
| **Поддомен (tenant)** | Каждая компания — отдельный `subdomain` в БД |
| **API-ключ** | `garp_...` с scope для чтения курсов/уроков/пользователей |
| **Мобильное приложение** | Flutter WebView или Capacitor — открывает ваш URL LMS |

## 3. Варианты подключения

### Вариант A — Поддомен GARPIUM (быстрый старт)

1. Мы создаём компанию в системе с полями:
   - `name` — название вашей компании
   - `subdomain` — например `acme` → `acme.lms.garpium.com`
   - `logoUrl` — URL или путь к вашему логотипу
   - `showPoweredBy: true` — подпись GARPIUM внизу (можно отключить по договорённости)

2. Вы настраиваете DNS:
   ```
   acme.lms.garpium.com  CNAME  →  lms.garpium.com
   ```

3. Пользователи заходят на поддомен, регистрируются, админ одобряет доступ.

### Вариант B — Ваш домен (полный white-label)

1. Вы указываете CNAME своего домена на наш сервер:
   ```
   learn.acme.com  CNAME  →  lms.garpium.com
   ```

2. Мы привязываем домен к вашей компании в middleware и SSL.

3. Внешне обучение полностью на вашем домене, GARPIUM указан только в футере.

### Вариант C — Partner API (встраивание в свой продукт)

Если у вас уже есть личный кабинет или CRM — можно **не открывать наш UI**, а забирать данные через API:

- Список курсов и уроков
- Контент урока (теория, задания)
- Создание пользователей из вашей системы

Ключ API создаётся в админке: `/admin/api`.

### Вариант D — PWA (браузер на телефоне)

Платформа работает в мобильном браузере. Пользователь может добавить сайт на главный экран через меню «Установить приложение» — откроется как полноэкранное веб-приложение без отдельной сборки APK/IPA.

## 4. Partner API

### Авторизация

Все запросы к `/api/v1/*` требуют заголовок:

```
Authorization: Bearer garp_xxxxxxxxxxxxxxxx
```

Ключ создаётся администратором GARPIUM в разделе **Админка → API**.

### Scopes (права)

| Scope | Что даёт |
|-------|----------|
| `read:courses` | Список курсов, модулей, уроков |
| `read:lessons` | Полный контент урока |
| `write:users` | Создание пользователей из вашей системы |

### Эндпоинты

#### GET `/api/v1/courses`

Список курсов с модулями и уроками.

```bash
curl -H "Authorization: Bearer garp_YOUR_KEY" \
  https://lms.garpium.com/api/v1/courses
```

Ответ:
```json
{
  "provider": "GARPIUM LMS",
  "company": "Ваша компания",
  "courses": [
    {
      "slug": "html",
      "title": "HTML",
      "modules": [
        {
          "title": "Старт",
          "lessons": [
            { "id": "h-01", "title": "Что такое HTML", "order": 1, "points": 10 }
          ]
        }
      ]
    }
  ]
}
```

#### GET `/api/v1/lessons/{lessonId}`

Полный урок: теория, секции, задание.

```bash
curl -H "Authorization: Bearer garp_YOUR_KEY" \
  https://lms.garpium.com/api/v1/lessons/h-01
```

#### POST `/api/v1/users`

Создание ученика (scope `write:users`):

```bash
curl -X POST \
  -H "Authorization: Bearer garp_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "firstName": "Иван",
    "lastName": "Петров",
    "email": "ivan@acme.com",
    "password": "secure-password"
  }' \
  https://lms.garpium.com/api/v1/users
```

Пользователь создаётся в вашей компании (по API-ключу). Статус — `PENDING`, пока админ не одобрит в `/admin`.

## 5. White-label: что настраивается

| Параметр | Поле в БД | Пример |
|----------|-----------|--------|
| Название компании | `Company.name` | Acme Corp |
| Поддомен | `Company.subdomain` | `acme` |
| Логотип | `Company.logoUrl` | `/uploads/acme-logo.png` |
| Цвет HTML-трека | `Company.brandHtml` | `#e8622a` |
| Цвет CSS-трека | `Company.brandCss` | `#0080ff` |
| Подпись GARPIUM | `Company.showPoweredBy` | `true` / `false` |
| Сайт компании | `Company.website` | `https://acme.com` |

При заходе на `acme.lms.garpium.com` middleware передаёт поддомен → подставляется ваш логотип в шапке.

## 6. Поток пользователя

```
Регистрация (/register)
    ↓
Ожидание одобрения (/pending)
    ↓
Админ одобряет (/admin)
    ↓
Личный кабинет (/dashboard)
    ↓
Карта курса (/courses/html) → Урок (/lessons/h-01)
    ↓
Прогресс сохраняется → Сертификат после 120 уроков
```

## 7. Развёртывание (для вашей инфраструктуры)

Если вы хотите хостить LMS у себя:

### Требования

- Node.js 20+
- PostgreSQL 14+
- Переменные окружения (`.env`):

```env
DATABASE_URL=postgresql://user:pass@localhost:5432/corporate_lms
NEXTAUTH_SECRET=случайная-строка-32-символа
NEXTAUTH_URL=https://lms.yourdomain.com
NEXT_PUBLIC_APP_URL=https://lms.yourdomain.com
```

### Команды

```bash
npm install
npx prisma migrate deploy
npx prisma db seed
npm run build
npm run start
```

Dev-сервер (Windows, путь с кириллицей):

```bash
npm run dev -- --webpack
```
